# Visual identity: CSS-custom-property tokens with a dual-theme cartographic system

Hexly's look is expressed as a **design-token layer of CSS custom properties**, not as per-component colours or a CSS-in-JS theme object. The tokens live in `apps/web/src/styles/` (`tokens.css`, `base.css`, `components.css`) and are pulled into the global `styles.css`. Every UI slice — grid, painting, content types, sharing — styles itself from _semantic_ tokens (`--ink`, `--gold`, `--surface`, `--terrain-forest`, `--space-4`) and from the shared component classes (`.panel`, `.btn`, `.tool`, `.chip`, `.coord`). A slice asks for a role, never a raw colour, so the whole app stays coherent and re-themes for free.

The identity is **"the cartographer's table, by starlight"** — one brand told at two hours of the day. **Parchment** (light) is an aged sea-chart: cream stock, sepia iron-gall ink, compass-gold, verdigris seas, burnt-sienna marginalia. **Astral** (dark) is the same chart under the night sky: midnight-indigo paper, a faint starfield and nebula, constellation-gold, aurora-teal. The two themes are deliberately _the same identity_: gold is the through-line (compass ink by day, constellation lines by night), teal carries the seas/aurora, and body text stays a warm parchment-cream even in the dark so the night theme never reads as a generic "dark mode." Type pairs **Cinzel** (engraved cartouche display), **Spectral** (literary serif body/UI), and **JetBrains Mono** (hex coordinates — the app's signature numeric detail).

We chose **the `[data-theme]` attribute on `<html>`** as the theming switch over a CSS-`@media`-only approach. The light palette lives on `:root`; `:root[data-theme='dark']` overrides it. A user's explicit choice must win over the OS preference (a GM may want the dark map on a bright screen), which an attribute supports and a pure `prefers-color-scheme` media query cannot. `ThemeService` owns the active theme, persists it to `localStorage`, and falls back to the OS preference when unset. A tiny inline script in `index.html` applies the stored/preferred theme **before first paint** to avoid a flash of the wrong theme.

Atmosphere is rendered in CSS, not images: a fixed body layer carries paper grain (light) or a tiled starfield plus nebula gradients (dark), and the canvas frame draws the infinite hex grid as a themed SVG **mask** so the grid ink tints with the theme. The living reference is the in-app **`/styleguide`** route, which renders the token layer back to the reader (palette, type scale, spacing, components) so downstream slices can see exactly what to adopt. Tokens are documented in `docs/design/identity.md`.
