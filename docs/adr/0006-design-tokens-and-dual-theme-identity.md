# Visual identity: CSS-custom-property tokens with a dual-theme cartographic system

Hexly's look is expressed as a **design-token layer of CSS custom properties**, not as per-component colours or a CSS-in-JS theme object. The tokens live in `apps/web/src/styles/` (`tokens.css`, `base.css`, `components.css`) and are pulled into the global `styles.css`. Every UI slice — grid, painting, content types, sharing — styles itself from _semantic_ tokens (`--ink`, `--gold`, `--surface`, `--terrain-forest`, `--space-4`) and the shared component classes (`.panel`, `.btn`, `.tool`, `.chip`, `.coord`). A slice asks for a role, never a raw colour, so the whole app stays coherent and re-themes for free. We chose this over per-component colours (drift, no single source of truth) and over a CSS-in-JS theme object (ties theming to the component runtime, when the same tokens must also drive the Canvas renderer and plain CSS).

Light and dark are deliberately **one identity at two times of day, not a light mode plus an unrelated dark mode** ("the cartographer's table, by starlight"). This is the binding constraint on the palette: a through-line carries across both themes (gold) and body text keeps the same warm tone in both, so the dark theme never reads as a generic "dark mode." Downstream slices' colour choices must honour it. The full palette, type pairing, and brand rationale live in `docs/design/identity.md` — that spec, not this ADR, is the place that evolves.

We chose **the `[data-theme]` attribute on `<html>`** as the theming switch over a CSS-`@media`-only approach. The light palette lives on `:root`; `:root[data-theme='dark']` overrides it. A user's explicit choice must win over the OS preference (a GM may want the dark map on a bright screen), which an attribute supports and a pure `prefers-color-scheme` media query cannot. `ThemeService` owns the active theme, persists it to `localStorage`, and falls back to the OS preference when unset. A tiny inline script in `index.html` applies the stored/preferred theme **before first paint** to avoid a flash of the wrong theme.
